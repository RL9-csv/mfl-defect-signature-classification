01풀어야 할 문제

검사 장비는 이미 진폭 임계(1.5V/26mm)로 "결함 유무"를 찍는다. 그래서 결함 유무를 또 맞히는 건 if문 한 줄(trivial)이라 의미가 없다.

→ 문제를 다시 정의했다: 임계를 넘은 결함이 "어떤 공간 형태"인가를 분류한다. 형태가 결함의 원인(root cause)을 가리키기 때문.

02핵심 통찰 — staircase

강철바가 회전(462rpm)하며 통과하므로, 하나의 결함이 센서 10개를 시간차를 두고 순차적으로 자극한다. 신호 위에서 결함이 대각선 궤적을 그린다 — 이게 staircase. (아래 애니메이션: 결함이 센서를 훑고 지나감)

A1A2A3A4A5B1B2B3B4B5

진폭이 모두 임계를 넘으므로 크기로는 못 가른다 → 오직 공간 패턴으로만 갈림 → 2D CNN의 필요성이 데이터로 성립

03공학적 사고 — 결정마다 근거

기법을 나열한 게 아니라, 막힐 때마다 가설을 세우고 데이터로 검증했다.

문제

정규화가 신호를 죽였다

채널별 robust z-score를 쓰자 결함 스파이크와 채널 간 상대크기가 눌려 macro-F1 0.337로 붕괴.

가설→검증: 스파이크를 살리는 log1p로 교체.

0.337 → 0.813
의심

장비 표기를 믿지 않았다

"센서가 진짜 어떤 순서로 붙어있나?" 30만 결함의 채널 전이를 집계해 배치를 역추정.

발견: A5–B1 인접, B5–A1=0 → 열린 사슬 → circular 아닌 replicate padding.

데이터로 topology 복원
검증

"진짜 공간을 쓰나?" 셔플 실험

채널 순서를 무작위로 섞어 재학습. 모델이 공간 배치를 쓴다면 무너져야 한다.

결과: 0.894 → 0.650, GB보다도 낮아짐 → 공간 의존 입증.

−0.24 폭락
절제

안 통한 것도 기록했다

명시적 방향벡터를 late-fusion으로 붙였더니 오히려 하락. 메타데이터(line speed 등)는 LOT 상수라 형태 판별력 0.

결론: CNN이 이미 충분 → 불필요한 것 제거(Occam).

도구 맹신 X, 검증 O
정합

지표와 손실을 일치시켰다

평가가 macro-F1인데 F1은 미분 불가. 확률로 TP/FP/FN을 연속화해 미분가능한 soft-F1을 직접 최적화.

효과: 약한 클래스(line/band) F1 개선.

0.903 → 0.906
정직

누수와 한계를 먼저 막았다

같은 LOT의 데이터가 train/test에 섞이면 누수 → LOT 단위 group split. 라벨은 규칙 기반 weak label임을 명시.

표현: "진짜 결함"이 아닌 "후보"로.
